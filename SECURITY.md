# Security Policy

## Supported Versions
Latest stable release of Printverse is actively supported.

## Reporting a Vulnerability
If you find a vulnerability, open a confidential Issue labeled 'security'.

Please include:
- Affected components/code/file
- Steps to reproduce (if possible)
- Impact level

## Project Security Guidelines
- Never commit `.env` or secrets to source
- Use strong authentication (JWT, password policy)
- Minimize PII and document retention

## Ethics & Privacy
- Design with user privacy and data minimization in mind
- Allow for pseudonym use when possible

Thank you for your help keeping the project secure!
